## 実施作業

- `apps/api/src/document-share-routes.test.ts` を追加し、複数ローカルユーザーの HTTP route 経由で以下を検証した。
  - direct readOnly ユーザーの `GET /documents/{id}/share` は 403。
  - direct readOnly ユーザーの `GET /documents` は対象文書だけ表示し、folder/group metadata を sanitize する。
  - owner は direct grants と tenantId を取得できる。
  - direct full ユーザーは元フォルダ full なしで share 更新できる。
  - direct full + destination folder full で move できる。
  - 移動先同名ファイルは 409、rename 指定で移動成功。
- `DocumentWorkspace.test.tsx` の 29 `it.todo` ブロックを削除した。
- `DocumentWorkspace.test.tsx` に readOnly 選択フォルダのフォルダ操作無効化、共有対象フォルダ権限による保存可否テストを追加した。
- task file に受け入れ条件、実装メモ、検証結果、未実施項目を記録した。
