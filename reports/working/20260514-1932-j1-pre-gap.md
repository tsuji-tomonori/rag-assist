## 3. 検討・判断したこと

- `GET /openapi.json` は `apps/api/src/app.ts` の runtime route として存在し、生成 Markdown は `apps/api/src/generate-openapi-docs.ts` が `app.request("/openapi.json")` から作るため、runtime JSON を source of truth とする現行方針は確認済みに分類した。
- `npm run docs:openapi:check` は summary / description / field description / authorization metadata の quality gate であり、REST / oRPC / shared contract drift 全体や generated Markdown stale を PR CI で fail する gateではないため、drift gate は missing / divergent を含む gap として整理した。
- `POST /documents` と同期 ingest は OpenAPI description で後方互換・非推奨が表現されている一方、機械可読 lifecycle metadata や `POST /chat` の runtime OpenAPI 上の compatibility 明示は不足しているため、API lifecycle は partially covered とした。
- generated OpenAPI Markdown は派生成果物であり手編集しない方針のため、`docs/generated/openapi*` は変更しなかった。
