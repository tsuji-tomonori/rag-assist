# PR #328 権限継承回帰修正 作業レポート

## 受けた指示

PR #328 のレビューで挙げられた 5 件の回帰テストを実装し、失敗したケースに基づいて API/Web の実装を修正し、再度テストで確認すること。

## 要件整理

- legacy `hasExplicitPolicy=false` child group は、既存保存フィールドが個別共有設定として解釈されるべき場合に親継承へ再解釈しない。
- `listDocumentGroups(user)` は Web が resource-level permission を判断できるよう、実効権限と継承元を返す。
- Web の新規子フォルダ作成で「親フォルダから継承」を選べるようにし、その場合は explicit policy payload を送らない。
- Web は global feature permission だけでなく、選択 folder の `effectivePermission` を見て write/delete/reindex/upload/share を抑止する。

## 検討・判断

- 既存データ互換性を優先し、`hasExplicitPolicy` が `false` として保存済みの child group は legacy explicit として扱う。一方、新規の親継承 child は `hasExplicitPolicy` を保存しないことで、`undefined` を inheritance の信号として扱う。
- API route schema は `visibility` を optional にして、Web が inherit 選択時に省略できるようにした。
- Web の resource-level guard は API の最終 403 を置き換えるものではなく、UI と handler の事前抑止として実装した。
- 追加したボタン title が既存 icon button と重複して Vitest の query を壊したため、操作用 icon button の title を残し、submit button の重複 title は外した。

## 実施作業

- `apps/api/src/rag/memorag-service.test.ts` に legacy shared child、legacy private child、実効権限注釈の回帰テストを追加した。
- `apps/api/src/folders/document-group-permissions.ts` に `resolveDocumentGroupPermissionDetail()` を追加し、`effectivePermission`, `policySource`, `inheritedFromFolderId` を導出できるようにした。
- `apps/api/src/rag/memorag-service.ts` で `listDocumentGroups()` の返却に実効権限注釈を加え、新規 inherit child では `hasExplicitPolicy` を保存しないようにした。

- `apps/web/src/features/documents/components/DocumentWorkspace.test.tsx` に inherit payload 省略と readOnly folder 操作抑止の回帰テストを追加した。
- `DocumentWorkspace` / `DocumentDetailPanel` / `useDocuments` を更新し、inherit 選択と selected folder の resource-level disabled を反映した。
