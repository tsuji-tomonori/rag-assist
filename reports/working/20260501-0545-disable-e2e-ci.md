# 作業完了レポート

保存先: `reports/working/20260501-0545-disable-e2e-ci.md`

## 1. 受けた指示

- e2e が不安定で落ちること、および CI 時間の増大による開発効率低下を背景に、e2e を CI から外して手動実行のみとする。

## 2. 要件整理

| 要件ID | 指示・要件 | 重要度 | 対応状況 |
|---|---|---:|---|
| R1 | e2e を CI 自動実行対象から外す | 高 | 対応 |
| R2 | e2e を手動実行のみで残す | 高 | 対応 |
| R3 | 変更をコミットし PR 文面を作成する | 高 | 対応予定 |

## 3. 検討・判断したこと

- 既存の e2e workflow は `pull_request` と `schedule` で自動実行されていたため、これらを外して `workflow_dispatch` のみ残せば「手動実行のみ」の要件を満たせると判断した。
- 既存の `if` 条件はイベント分岐用だったため、手動実行専用化に合わせて削除し、workflow 定義を簡素化した。

## 4. 実施した作業

- `memorag-bedrock-mvp/.github/workflows/e2e.yml` のトリガーを `workflow_dispatch` のみに変更。
- `smoke` / `nightly` ジョブの event 条件 (`if`) を削除。
