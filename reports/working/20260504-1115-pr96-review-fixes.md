## 3. 検討・判断したこと

- `needsSearch=false` だけで retrieval をスキップせず、`canAnswerFromQuestionOnly` と `executeComputationTools` 後の usable `computedFacts` 確認を組み合わせる方針にした。
- `asOfDate` は public API request schema には追加せず、内部 `ChatInput` の benchmark / test 用フィールドとして追加した。
- 未対応系 computed fact は回答として説明可能なため answerable は維持しつつ、`calculation_unavailable` / `structured_index_unavailable` の専用 reason で通常の `sufficient_evidence` と区別した。
- `nowIso` の server path は UTC ISO のままとし、docs で `today` / `timezone` と役割を分けて説明した。

## 4. 実施した作業

- `ToolIntent` に `canAnswerFromQuestionOnly` を追加し、compute-only path を usable `computedFacts` がある場合に限定した。
- computed fact が空の false positive では通常の RAG retrieval へ戻るよう graph を修正した。
- `ChatInput.asOfDate` / `asOfDateSource` を追加し、`build_temporal_context` に渡した。
- `current_date` computed fact を追加し、「今日の日付は？」に対応した。
- `days_until` の過去日は `deadline_status` に寄せ、超過日数を返すよう修正した。
- 金額計算の期間抽出から単独 `年` / `月` を外し、safe integer range 超過時は unavailable にした。
- prompt、architecture view、詳細設計をレビュー指摘に合わせて更新した。
- fallback、asOfDate、current date、past date、calendar date 混入、不正 `usedComputedFactIds` のテストを追加した。
