# PR332 回答送信スコープ修正レポート

## 受けた指示

PR #332 の再レビューで blocking 指摘された、担当外 `ANSWER_EDITOR` が `POST /questions/{questionId}/answer` で回答送信できる認可不整合を修正する。

## 要件整理

- `answer:publish` は route の機能権限として維持する。
- ticket resource scope は、担当者本人、担当グループ所属、または `support:ticket:read:all` を持つ全件管理者に限定する。
- 担当外ユーザーには 404 を返し、ticket 本文や internal field を返さない。

## 実施作業

- `POST /questions/{questionId}/answer` の判定を `canReadAllTickets(user) || canAccessAssignedTicket(user, question)` に変更した。
- 担当外 `ANSWER_EDITOR`、担当 user、担当 group、全件管理者の route unit test を追加した。
- 担当外 404 response に `title`, `question`, `internalMemo`, `sanitizedDiagnostics` が含まれないことを固定した。

## Fit 評価

blocking 指摘の回答送信 route を担当者スコープへ合わせた。管理者の全件対応は `support:ticket:read:all` 経由に限定され、通常 `ANSWER_EDITOR` の担当外回答はできない。

## 未対応・制約・リスク

- non-blocking 指摘の inaccessible favorite 作成拒否は今回の blocking 修正範囲外として未対応。
- full CI matrix はローカルでは全件再実行していない。変更範囲に対する API route test / lint / typecheck / OpenAPI check を選択実行した。
