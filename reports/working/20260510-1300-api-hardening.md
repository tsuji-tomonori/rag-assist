## 3. 検討・判断したこと

- production では `AUTH_ENABLED=true`、Cognito 設定、`DOCS_BUCKET_NAME`、`CORS_ALLOWED_ORIGINS` を起動時に検証し、設定漏れを早期に落とす方針にした。
- local/test の開発体験は維持し、production 以外では `AUTH_ENABLED=false` と `CORS_ALLOWED_ORIGINS=*` の既定を残した。
- `protectedApiPaths` は public allowlist に置き換え、非 public path は route 追加時に原則認証 middleware を通る構成にした。
- `looseRoute` の `any` は削除したが、`@hono/zod-openapi` の深い response 型展開で typecheck が過剰に重くなる route があったため、route helper の戻り値は `RouteConfig` に緩めた。handler 側の `(c.req as any)` は `validJson` / `validParam` に局所化した。
- REST schema と oRPC contract の完全統合は blast radius が大きいため、今回は README / API design に drift 確認 gate を明記した。

## 4. 実施した作業

- `config.ts` に production fail-closed validation と `CORS_ALLOWED_ORIGINS` を追加。
- `app.ts` で CORS allowlist、public allowlist 認証 middleware、JSON `HTTPException`、JSON `notFound` を実装。
- `AppEnv` と `MiddlewareHandler<AppEnv>` を追加し、Hono context variables を明示。
- route handler の `(c.req as any).valid(...)` を `validJson` / `validParam` に置換。
- typed route 化で露出した response schema のズレを修正。
- access-control policy test を public allowlist 前提へ更新。
- fail-closed config、JSON error contract、CORS header の contract test を追加。
- README、Operations、API design docs に運用・契約変更と REST / oRPC drift 確認方針を追記。
