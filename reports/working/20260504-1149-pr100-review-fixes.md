## 3. 検討・判断したこと

- この PR では Cognito をユーザー発見用 directory、管理台帳を管理画面上のロール、停止、再開、削除状態の source of truth として扱う方針に固定した。
- 既存のロール付与 API は Cognito group を変更しないため、表示上の管理ロールと JWT による実効権限が異なり得る点を docs に明記した。
- Cognito group 取得は一覧表示の補助情報なので、1ユーザーの失敗で `/admin/users` 全体を 500 にせず、該当ユーザーだけ `groups: []` として継続する方針にした。

## 4. 実施した作業

- `CognitoUserDirectory` の group 取得を concurrency 5 に制限した。
- `AdminListGroupsForUser` のユーザー単位の失敗を捕捉し、警告ログと `groups: []` で継続するようにした。
- `ListUsers` pagination、`AdminListGroupsForUser` pagination、一部失敗、並列数制御の adapter test を追加した。
- service test に email merge、管理台帳ロール維持、停止状態維持、deleted tombstone 非表示の確認を追加した。
- `auth.ts` を `config.authEnabled` / `config.cognitoRegion` / `config.cognitoUserPoolId` / `config.cognitoAppClientId` に寄せた。
- README と OPERATIONS に、Cognito directory と管理台帳 source of truth の役割分担を追記した。
