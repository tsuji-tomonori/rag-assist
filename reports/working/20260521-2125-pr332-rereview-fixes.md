# PR332 再レビュー指摘対応レポート

## 受けた指示

PR #332 の再レビューで残った mergeability、default support assignee group、履歴 favorite 表示順、Favorite 再保存時の既存値保持を修正し、検証・PR 更新まで進める。

## 要件整理

- `origin/main` を取り込み、PR の `mergeable=false` 要因を解消する。
- CDK 本番環境の API / Heavy API / worker に `DEFAULT_SUPPORT_ASSIGNEE_GROUP_ID` を設定する。
- default support group が `ANSWER_EDITOR` のような role 名に見える場合でも、担当者 Query の group scope から落とさない。
- 会話履歴は favorite 付与後に favorite 優先 sort と最終 limit を適用する。
- Favorite 再保存では既存 `createdAt` と未指定 `note` / `label` を保持する。

## 実施作業

- `origin/main` を merge し、生成済み web inventory の競合を再生成で解消した。
- CDK context `defaultSupportAssigneeGroupId` を追加し、既定値を `ANSWER_EDITOR` として API runtime env に配布した。
- contract の `ApiRuntimeEnv` に `DEFAULT_SUPPORT_ASSIGNEE_GROUP_ID` を追加した。
- `supportGroupIds()` が configured default group を application role filter で除外しないようにした。
- `createQuestion()` の default group 参照を config 経由に統一した。
- ConversationHistory store の事前 20 件制限を外し、service 側で favorite enrichment 後に sort / limit するようにした。
- `DynamoDbFavoriteStore.save()` を `UpdateItemCommand` に変更し、`if_not_exists` と明示 field 更新で既存値を保持するようにした。
- 関連 unit test、infra template test、snapshot、生成 inventory、運用 docs を更新した。
