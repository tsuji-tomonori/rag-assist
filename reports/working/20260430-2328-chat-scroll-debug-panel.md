## 1. 受けた指示

- 主な依頼: チャットが長くなったとき、デバッグパネルまで下に押し流されないようにし、チャット部分だけでスクロールできるようにする。
- 追加要件: チャットを手動スクロールしなくても、新しいチャットが都度上部に表示されるよう自動スクロール機能を入れる。
- 成果物: Web フロントエンドのレイアウトと自動スクロール実装。

## 3. 検討・判断したこと

- デバッグパネルが下に移動する原因は、ワークスペース全体がチャット本文の高さに合わせて伸びる構造と判断した。
- 画面全体とメイン領域を viewport 高に固定し、`message-list` のみ `overflow-y: auto` にする方針を採用した。
- 新しいメッセージと処理中メッセージに ref を付け、追加時に `scrollIntoView({ block: "start" })` を呼ぶことで、最新チャットがスクロール領域の上部に来るようにした。
- 既存テストの mock が `/questions` の空レスポンスを返すケースに合わせ、質問一覧 API は未定義時に空配列へフォールバックするようにした。

## 4. 実施した作業

- `App.tsx` に最新メッセージ用 ref と自動スクロール effect を追加。
- 最新の通常メッセージおよび処理中メッセージにスクロール対象 ref を付与。
- `styles.css` でアプリ全体、メイン領域、ワークスペース、チャットカードの高さと overflow を調整。
- `message-list` のみ縦スクロールするよう設定し、スクロールの伝播を抑制。
- `api.ts` の `listQuestions` を空配列フォールバック付きに修正。
- Web の型チェックとテストを実行。
- Web dev server を起動。
