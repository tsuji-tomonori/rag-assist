## 3. 検討・判断したこと

- デバッグ履歴自体は残しつつ、現在の会話に紐づくトレースだけを標準表示するようにした。履歴を消すと過去実行の選択用途を壊すため、表示選択のリセットで対応した。
- 空状態はメッセージ配列に入れず、メッセージがないときだけ描画するビューにした。これにより「新しい会話」後にデフォルトチャットが履歴として残らない。
- ファイル入力は DOM 上の選択状態が残りやすいため、会話キーで再マウントして確実にクリアするようにした。
- 処理中表示はチャット欄とデバッグパネルの両方に出し、スピナーとドットアニメーションで待機中であることを明示した。

## 4. 実施した作業

- `App.tsx` に pending 状態を追加し、送信中のチャット表示とデバッグパネル表示を分離した。
- `selectedTrace` の既定値から `debugRuns[0]` へのフォールバックを外し、新規会話時に過去トレースが出ないようにした。
- 「新しい会話」でメッセージ、入力、添付、エラー、実行ID、ステップ展開状態、処理中状態をリセットするようにした。
- 初回・新規会話の空状態として、アイコン、資料件数、候補プロンプトを表示する UI を追加した。
- CSS にスピナー、ドットアニメーション、処理中デバッグパネル、空状態のスタイルを追加した。
- UI 状態遷移のテストを追加・更新した。
