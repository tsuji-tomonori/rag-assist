## 3. 検討・判断

- Web の `SignUp` / `ConfirmSignUp` と画面は既存だった一方、CDK は `selfSignUpEnabled: false` で handler が未接続だったため、GAP-RD-020 の本体を infra 配線と判断した。
- 既存 handler は環境変数から任意の canonical application role を選べたため、self-sign-up で上位 role を選べないよう `CHAT_USER` をコード上の固定値にした。
- User Pool と Lambda IAM の循環依存を避けるため、IAM resource は同一 account/region の `userpool/*` とし、action は `AdminAddUserToGroup` のみに限定した。group 境界は handler の固定値で多層化した。
- 一時障害は 3 回の bounded retry、継続障害は例外伝播とし、未付与を成功扱いしない。
- local integration test は実 AWS の確認メール、service retry、throttling を再現しないため、実 AWS E2E は未検証として残した。

## 4. 実施作業

- User Pool self-sign-up を有効化し、post-confirmation Lambda、ログ、IAM、Cognito trigger を追加した。

- duplicate invocation、対象外 trigger、入力不備、一時障害、継続障害の unit test を追加した。
- Web auth integration test に sign-up、confirmation、sign-in、`CHAT_USER` token、認証付き documents API 呼び出しを追加した。

- `FR-025`、`DES_DLD_004`、requirements baseline、coverage trace、CDK snapshot、infra inventory を更新した。
