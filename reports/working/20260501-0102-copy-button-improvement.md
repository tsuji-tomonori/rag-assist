## 3. 検討・判断したこと

- 現状はテキストのみのボタンで、押下後の状態変化が弱かったため、視認性の高いアイコン＋状態表示を追加する方針を採用。
- コピー不可な空文字ケースは disabled 化して誤操作を減らす判断を行った。
- 改修範囲は `AssistantAnswer` と関連 CSS に限定し、既存 API やデータ構造への影響を避けた。

## 4. 実施した作業

- `copy` アイコンを Icon 定義へ追加。
- コピーアクションボタンをアイコン付きに変更し、コピー済み状態で `check` アイコンと「コピー済み」表示を出すよう変更。
- 空のプロンプト/回答時にボタンを disabled 化。
- コピー済み状態・disabled 状態のスタイルを追加。
