## 要件

- FR-021: 利用者は回答不能な質問を担当者向け問い合わせとして登録できること。

## 受け入れ条件（この要件専用）

- AC-FR021-001: 利用者は問い合わせのタイトル、質問本文、依頼者情報、担当部署、優先度を送信できること。
- AC-FR021-002: システムは登録済み問い合わせを `open`、`answered`、`resolved` の状態で管理できること。
- AC-FR021-003: 担当者は問い合わせに回答本文と内部メモを保存できること。
- AC-FR021-004: 通常利用者が問い合わせを送信した後に、担当者一覧や debug trace 一覧の取得に依存せず送信完了を表示できること。

## 要件の源泉・背景

- 源泉: `reports/working/20260501-0034-assignee-answer-ui-ux.md`、`reports/bugs/20260502-1135-question-escalation-forbidden.md`、現行 `/questions` API 実装。
- 背景: RAG が根拠不足で回答できない質問を、人手確認の業務フローへ引き継げるようにするため。

## 要件の目的・意図

- 意図: RAG の回答不能を利用者の行き止まりにせず、担当者対応へエスカレーションできる状態にする。
- 意図: 通常利用者、担当者、管理者の権限境界を保ちながら問い合わせ導線を成立させる。
