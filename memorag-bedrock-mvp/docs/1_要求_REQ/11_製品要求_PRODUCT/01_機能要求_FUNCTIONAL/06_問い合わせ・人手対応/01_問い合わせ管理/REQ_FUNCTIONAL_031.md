## 要件

- FR-031: 担当者は問い合わせをカンバン形式で状態別に確認できること。

## 受け入れ条件（この要件専用）

- AC-FR031-001: 担当者向け画面は問い合わせを `未対応`、`対応中`、`確認待ち`、`解決済み` の列で表示できること。
- AC-FR031-002: `open`、`answered`、`resolved` の既存 ticket 状態を破壊せず、画面表示用の列へ分類できること。
- AC-FR031-003: 問い合わせカードはタイトル、優先度、担当部署、依頼者、受付日時を表示できること。
- AC-FR031-004: 担当者はカンバン上の問い合わせカードを選択して詳細を確認できること。
- AC-FR031-005: モバイル幅ではカンバン列と周辺 UI が 1 列に畳まれること。

## 要件の源泉・背景

- 源泉: `reports/working/20260506-1940-assignee-kanban-ui.md`、`reports/working/20260506-1947-assignee-kanban-acceptance-review.md`、`tasks/done/20260506-1947-assignee-kanban-flow-correction.md`。
- 背景: 問い合わせ件数が増えた場合、単一の一覧だけでは担当者が未対応、対応中、確認待ち、解決済みを把握しにくい。
- 背景: PR #128 で担当者対応 UI をカンバン形式へ変更し、既存 API 契約を維持したまま表示分類を追加した。

## 要件の目的・意図

- 意図: 担当者が問い合わせの対応状況を視覚的に把握し、優先度と担当部署を見ながら処理できるようにする。
- 意図: UI の状態分類を追加しても、API や store の既存状態契約を不必要に増やさない。
