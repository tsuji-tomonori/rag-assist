## 要件

- FR-035: 会話履歴は問い合わせ結果の状態を利用者に通知できること。

## 受け入れ条件（この要件専用）

- AC-FR035-001: 会話履歴は担当者回答が返った問い合わせを `返答あり` として識別できること。
- AC-FR035-002: 会話履歴は未解決の問い合わせを `確認待ち` として識別できること。
- AC-FR035-003: 会話履歴は解決済みの問い合わせを `解決済み` として識別できること。
- AC-FR035-004: 履歴またはお気に入り表示中は、未解決 ticket の状態を継続的に同期できること。
- AC-FR035-005: 通知は対象利用者自身の会話履歴に紐づく問い合わせに限定されること。
- AC-FR035-006: 通知のための同期は内部メモ、debug trace、担当者向け一覧を通常利用者へ公開しないこと。

## 要件の源泉・背景

- 源泉: `reports/working/20260504-1120-question-answer-notifications.md`、`reports/working/20260504-1147-review-feedback-history-polling.md`。
- 背景: 利用者が問い合わせ後に担当者回答の到着を見落とさないよう、履歴に `返答あり`、`確認待ち`、`解決済み` のバッジを追加した。
- 背景: 通知はリアルタイム push ではなく、会話や履歴に紐づいた ticket ID の targeted GET と polling による UI 同期として整理されている。

## 要件の目的・意図

- 意図: 利用者が過去会話から問い合わせ結果を見つけやすくする。
- 意図: 回答到着通知を本人履歴に閉じ、問い合わせ一覧や debug trace の権限境界を越えない。
- 意図: push 通知がない段階でも、画面内の状態同期で回答到着を分かるようにする。
