## 背景

実作業前の task file と受け入れ条件が不足すると、完了条件、検証結果、PR コメントの対応を後から追跡しづらくなる。

## 目的

- PRJ-003: MemoRAG MVP の実作業は、実装前に task file と検証可能な受け入れ条件を作成し、状態に応じて `tasks/todo/`、`tasks/do/`、`tasks/done/` で管理しなければならない。

## 制約

- PRJ-003-C-001: 実作業前に、作業 task を `tasks/do/` に Markdown で作成しなければならない。
- PRJ-003-C-002: 作業 task には、背景、目的、対象範囲、方針、必要情報、実行計画、ドキュメントメンテナンス計画、受け入れ条件、検証計画、PR レビュー観点、リスクを記載しなければならない。
- PRJ-003-C-003: 作業 task の受け入れ条件は、実装前に検証可能な条件として記載しなければならない。
- PRJ-003-C-004: PR への受け入れ条件確認コメント後、完了条件を満たした task は `tasks/done/` へ移動し、状態を `done` に更新しなければならない。

## 受け入れ条件

- PRJ-003-AC-001: 実作業前に task file が `tasks/do/` に存在すること。
- PRJ-003-AC-002: task file に必要セクションと検証可能な受け入れ条件が記載されていること。
- PRJ-003-AC-003: PR 受け入れ条件確認コメント後、完了した task file が `tasks/done/` に移動され、状態が `done` または同等に更新されていること。
- PRJ-003-AC-004: 未達または未検証の受け入れ条件がある場合、task file が完了扱いになっていないこと。
