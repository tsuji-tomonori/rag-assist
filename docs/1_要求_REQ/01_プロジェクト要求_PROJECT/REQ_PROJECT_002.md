## 背景

元 worktree の未追跡変更または未コミット変更を実作業に混ぜると、依頼範囲、検証対象、PR 差分の境界が不明確になる。

## 目的

- PRJ-002: MemoRAG MVP の実作業は、元 worktree の未追跡変更または未コミット変更を混入させない専用 worktree で実施しなければならない。

## 制約

- PRJ-002-C-001: 実作業を伴う依頼は、原則として `origin/main` から作成した専用 worktree で実施しなければならない。
- PRJ-002-C-002: 専用 worktree は、元 worktree の未追跡変更または未コミット変更を混入してはならない。

## 受け入れ条件

- PRJ-002-AC-001: 実作業の branch が、原則として `origin/main` 由来の専用 worktree で作成されていること。
- PRJ-002-AC-002: PR 差分に、元 worktree の無関係な未追跡変更または未コミット変更が含まれていないこと。
- PRJ-002-AC-003: worktree を省略した場合は、task、report、PR 本文、または最終回答に理由が記録されていること。
