# 要件定義（1要件1ファイル）

- 要件ID: `SQ-001`
- 種別: `REQ_SERVICE_QUALITY`
- 状態: Draft
- 優先度: S

## 要件

- SQ-001: RAG品質改善のリリース判断では、不回答性能、根拠性、検索再現率、権限外漏えい、レイテンシを継続測定すること。

## 受け入れ条件（この要件専用）

- AC-SQ001-001: benchmark summary に `abstentionRecall` または同等の不回答 recall が含まれること。
- AC-SQ001-002: benchmark summary に unsupported answer rate または faithfulness 系指標が含まれること。
- AC-SQ001-003: benchmark dataset は権限外文書を期待しない `must-not-access` 相当の観点を表現できること。
- AC-SQ001-004: benchmark summary に p95 latency または同等の遅延指標が含まれること。
- AC-SQ001-005: 閾値を決めていない指標は Draft として扱い、合格済みとは記録しないこと。

## 要件の源泉・背景

- 源泉: ユーザー提示の評価方針、現行 `benchmark/run.ts` の実装確認。
- 背景: RAG改善では、検索で取れていない情報を LLM で安定補完できないため、retrieval と groundedness を分けて測る必要がある。

## 要件の目的・意図

- 目的: リリース判断と改善優先度を測定可能にする。
- 意図: 回答できる件数だけでなく、答えない品質と漏えい防止を同時に管理する。
- 区分: サービス品質制約。
