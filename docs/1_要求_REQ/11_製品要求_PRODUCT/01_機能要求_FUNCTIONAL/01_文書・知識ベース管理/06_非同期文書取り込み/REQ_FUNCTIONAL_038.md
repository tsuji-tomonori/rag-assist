## 要件

- FR-038: 文書取り込みが同期 API の実行時間内に完了しない可能性があるとき、システムは利用者または runner が後から状態を確認できる非同期取り込み run として処理できること。

## 受け入れ条件（この要件専用）

- AC-FR038-001: 文書 upload session から非同期取り込み run を開始できること。
- AC-FR038-002: run 作成者または許可された管理者は、run の状態を取得できること。
- AC-FR038-003: run 作成者または許可された管理者は、run の進捗 event を購読または取得できること。
- AC-FR038-004: 成功した run は、文書が QA 利用可能になったことを確認できる manifest または summary を返すこと。
- AC-FR038-005: 失敗した run は、同期 API timeout と区別できる失敗状態と理由を返すこと。
- AC-FR038-006: 通常運用で大きな PDF、OCR fallback、embedding が絡む文書を扱う場合、同期 ingest API だけに依存しない導線があること。

## 要件の源泉・背景

- 源泉: `reports/working/20260507-2035-async-ingest-runs.md`
- 背景: 大きな PDF や OCR fallback では、同期 API の timeout 内に取り込みが完了しない可能性がある。
- 背景: UI と benchmark runner の両方で、取り込み処理の進捗と最終状態を後から確認できる必要がある。

## 要件の目的・意図

- 目的: 文書取り込みの実行時間が長い場合でも、利用者や runner が処理結果を追跡できるようにする。
- 意図: 同期 API の timeout 延長だけに依存せず、文書登録と QA 利用可能化の間の状態を観測可能にする。
- 区分: 機能要求。
