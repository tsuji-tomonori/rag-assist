## 要件

- FR-001: 文書登録権限を持つ利用者は、管理可能な保存先を用意または選択し、単一の文書をその保存先へアップロードできること。

## 受け入れ条件（この要件専用）

- AC-FR001-001: 管理可能な保存先が存在する場合、利用者は文字ラベル付きの `ドキュメントを追加` から保存先を選択し、文書登録を開始できること。
- AC-FR001-002: 管理可能な保存先がなくフォルダ作成権限がある場合、利用者は `新しいフォルダ名（必須）` だけを入力してルート直下へ非公開フォルダを作成でき、作成 request に `name` 以外の任意項目を含めないこと。
- AC-FR001-003: 簡易フォルダ作成に成功した場合、作成応答のフォルダを保存先へ選択し、作成済み通知と次の手動ファイル選択へ focus を提示すること。非同期処理完了後に file picker を自動起動しないこと。
- AC-FR001-004: 保存先候補には `effectivePermission=full` の実データ由来フォルダだけを表示し、UI の候補制御とは別に API の認可判定を維持すること。
- AC-FR001-005: 利用者は選択済み保存先を確認し、1ファイルを選択してアップロード要求を送信できること。
- AC-FR001-006: フォルダ0件かつ文書0件、選択フォルダ内0件、絞り込み結果0件を区別し、それぞれ `ドキュメントを追加` または `条件をクリア` の実行可能な回復導線を表示すること。
- AC-FR001-007: 文書登録権限も管理可能な保存先もフォルダ作成権限もない場合、架空の保存先を表示せず、操作不能理由を明示すること。
- AC-FR001-008: 文書追加 dialog は操作順、必須入力、保存先、成功・失敗状態に accessible name または description を持ち、キーボードで閉じられ、320 CSS px 幅で通常ページの横スクロールなしに操作できること。

## 要件の源泉・背景

- 源泉: 旧 `docs/1_要求_REQ/README.md` に定義された原子要件を分割移管。
- 源泉: 2026-07-13 のドキュメント管理画面確認と、`CreateDocumentGroupRequestSchema`、`DocumentGroup.effectivePermission`、`FR-076` の既存契約。
- 背景: 要求単位で差分レビュー・追跡・変更管理をしやすくするため、1要件1ファイル運用へ移行。
- 背景: 保存先未選択時に主操作が無効となり、名称以外の任意フォルダ設定が初回登録の必須入力に見えるため、最短登録フローを明文化する必要がある。

## 要件の目的・意図

- 目的: 利用者が知識源となる文書を登録できる入口を確保する。
- 意図: 初期状態から「保存先を用意」「ファイルを選択」の順で登録を完了し、詳細な共有・管理設定は後から行えるようにする。
- 区分: 機能要求として管理し、関連要件と独立に改訂可能な単位を維持する。
