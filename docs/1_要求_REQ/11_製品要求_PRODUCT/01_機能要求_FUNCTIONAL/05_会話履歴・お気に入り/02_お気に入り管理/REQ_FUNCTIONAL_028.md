## 要件

- FR-028: 利用者は自身の会話履歴をお気に入り登録し、後でお気に入りのみを抽出して再表示できること。

## 受け入れ条件（この要件専用）

- AC-FR028-001: 利用者は会話履歴一覧で任意の会話をお気に入りに追加または解除できること。
- AC-FR028-002: お気に入り状態は会話履歴 item とは別の shortcut として保存され、同じ利用者のお気に入り再取得後も維持されること。
- AC-FR028-003: 会話履歴一覧ではお気に入り会話が通常会話より優先して表示されること。
- AC-FR028-004: 利用者はお気に入りのみの一覧を表示できること。
- AC-FR028-005: お気に入り状態は本人の会話履歴保存境界に閉じ、別 userId の履歴として返さないこと。

## 要件の源泉・背景

- 源泉: ユーザーからの「お気に入り機能を実装して」という追加依頼。
- 背景: 既存 UI には会話履歴の検索、並び替え、削除があり、再利用したい会話を素早く探す導線が不足していた。
- 背景: 会話履歴以外の文書、フォルダ、プリセットなども shortcut 化できるよう、`/favorites` と専用 store に分離する。

## 要件の目的・意図

- 目的: 利用者が重要な会話を履歴一覧からすぐ再開できるようにする。
- 意図: お気に入りは対象本体ではなく shortcut として扱い、解除時に会話、文書、フォルダなどの本体を削除しない。
- 意図: お気に入り操作によって会話本文や回答内容の権限境界を広げない。
- 区分: 機能要求。
