# Issue #345 アプリ shell の skip link を実装する

状態: done

タスク種別: 修正

## 背景

Issue #345 と `SQ-016` は keyboard-only 操作時の skip/recovery を要求する。PR #385 は navigation landmark、contrast、target size、PR #396 は primary navigation の keyboard journey を検証したが、PR #420 final head `f6a0e923` の認証後 shell には反復 navigation を迂回する skip link が存在しない。

## 目的

認証済み利用者が keyboard-only で rail navigation と top bar を毎回通過せず、主コンテンツへ直接移動できる native skip link と自動回帰証跡を追加する。

## 対象範囲

- `AppShell` の landmark 構造と skip link
- skip link の focus-visible 表示
- desktop / 320px Chromium の keyboard E2E
- `SQ-016`、UI design、trace manifest、generated Web inventory の同期

対象外:

- 既存 PR #396 の primary navigation keyboard journey の再実装
- representative screen reader、実機、Firefox、WebKit の実測
- login 前画面、dialog focus trap、各 feature 内の heading navigation

## 軽量なぜなぜ分析

### 問題文

2026-07-17 時点の PR #420 final head では、認証後の `AppShell` に skip link がなく、keyboard-only 利用者は主コンテンツへ到達するたびに反復 navigation controls を順に通過する必要がある。`AC-SQ016-002` の skip 条件を自動検証する証跡もない。

### 確認済み事実

- `apps/web/src/app/AppShell.tsx` は `RailNav` の前に focusable な bypass control を描画しない。
- outer `main.app-frame` が `RailNav` と `.main-area` の両方を包含し、主コンテンツだけを指す focus target がない。
- repository search で production UI / E2E に `skip-link`、`メインコンテンツへ移動`、focus target の実装が見つからない。
- `AC-SQ016-002` は keyboard focus と skip/recovery を要求する。
- #396 は rail navigation の Tab/Enter/Space journey であり、反復 block の迂回を検証しない。

### 推定・未確認

- representative screen reader が current shell をどう読み上げるかは未実測である。
- 実 browser ごとの fragment focus behavior は Chromium 以外で未確認である。

### 根本原因

認証後 shell の layout container、navigation landmark、main landmark、keyboard focus target が分離されておらず、`AC-SQ016-002` の skip 条件を具体化した component/E2E gate がなかった。

### 是正方針

- layout root を非 landmark container にし、主コンテンツ領域を一意な `main` landmark/focus target にする。
- rail より前に native anchor の skip link を置き、通常時は画面外、focus-visible 時は viewport 内へ表示する。
- desktop / 320px で最初の Tab、表示、Enter、URL、focus、main landmark 一意性を E2E 固定する。
- 正規要件を変更せず、design/trace/generated evidence を同期する。
