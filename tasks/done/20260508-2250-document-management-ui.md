# ドキュメント管理UIを参照画像に寄せる

## 背景

ユーザーから、参照画像のような「社内QAチャットボットエージェント」のドキュメント管理画面にしてほしいという依頼があった。

## 目的

既存 Web UI のドキュメント画面を、フォルダツリー、ファイル一覧、詳細/共有パネルを備えた業務向けドキュメント管理画面に近づける。

## スコープ

- 対象: `memorag-bedrock-mvp/apps/web` のドキュメント画面
- 含む: レイアウト、表示文言、モック/既存データ表示、スタイル、必要なテスト更新
- 含まない: 新規 API、永続化仕様、実ファイル操作のバックエンド実装

## 作業計画

1. 既存の `DocumentWorkspace`、レイアウト、スタイル、テストを確認する。
2. 参照画像の主要構成に合わせてドキュメント画面を実装する。
3. 既存テストを更新し、Web UI の targeted 検証を実行する。
4. 作業レポートを `reports/working/` に作成する。

## ドキュメントメンテナンス方針

今回の変更は Web UI の見た目と画面構成に限定する。API、運用手順、環境変数に変更がない場合は durable docs は更新せず、作業レポートに判断理由を残す。

## 受け入れ条件

- [x] ドキュメント画面が参照画像に近い左フォルダツリー、中央ファイル一覧、右詳細パネルの構成になる。
- [x] 上部のモデル、ドキュメント、実行ID、デバッグモード、新しい会話の操作領域と整合する。
- [x] ファイル一覧にチェック、ファイル種別、更新日時、チャンク数、メモリ、再インデックス、操作メニューが表示される。
- [x] 右パネルにフォルダ情報、共有設定、フォルダ操作、最近の更新が表示される。
- [x] 既存の Web UI テストまたは対象コンポーネントテストが更新され、必要な検証が実行される。
- [x] 作業レポートが `reports/working/` に作成される。

## 検証計画

- `npm --prefix memorag-bedrock-mvp run test -w @memorag-mvp/web -- DocumentWorkspace`
- `npm --prefix memorag-bedrock-mvp run typecheck -w @memorag-mvp/web`
- `git diff --check`

## PRレビュー観点

- 参照画像の主要情報設計を過不足なく反映していること。
- UI変更が既存の認可境界や API 契約を変更していないこと。
- 実データ未連携の操作を実行済みの機能として誤認させないこと。

## リスク

- 参照画像は単一画面のため、レスポンシブ時の期待挙動は推定で補う。
- 既存 API にフォルダ/共有概念がない場合、表示はフロントエンド内の画面用データに限定される。
