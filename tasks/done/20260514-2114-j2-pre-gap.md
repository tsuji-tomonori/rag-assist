## 背景

Wave 5-pre `J2-pre-gap` worker として、仕様 14A/14D 章と現実装の `apps/api/src/app.ts`、debug feature、worker runId 周辺のギャップと影響範囲を調査し、後続 `J2-debug-4tier-and-middleware` の scope/scope-out を整理する。

別 worker が I-pre を並行実施するため、他 worktree の変更や元 worktree の未追跡ファイルを取り込まず、専用 worktree 上で完結させる。

## 目的

- `docs/spec/2026-chapter-spec.md` の 14A/14D と現実装の一致・不足・逸脱を確認する。
- debug panel の既存表示項目・権限境界、public endpoint/auth middleware/CORS/SSE/worker runId の安全境界を踏襲対象として記録する。
- 後続実装で扱う scope と scope-out を `docs/spec/gap-phase-j2.md` に明示する。

## 範囲

- 仕様: `docs/spec/2026-chapter-spec.md` の 14A/14D。
- API: `apps/api/src/app.ts`, `apps/api/src/routes/debug-routes.ts`, `apps/api/src/chat-run-worker.ts`, `apps/api/src/security/access-control-policy.test.ts`。
- Web: `apps/web/src/features/debug/`。
- 周辺: CORS、public endpoint、auth middleware、SSE Last-Event-ID、worker runId 関連実装。
- Docs: `docs/spec/gap-phase-j2.md`, `docs/spec/CHAPTER_TO_REQ_MAP.md`, 必要時 `docs/spec-recovery/08_traceability_matrix.md`。
