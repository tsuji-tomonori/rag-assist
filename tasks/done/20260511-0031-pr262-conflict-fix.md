## なぜなぜ分析サマリ

- 問題文: PR #262 が GitHub 上で `CONFLICTING / DIRTY` になり、main へ merge できない状態になった。
- 確認済み事実:
  - `gh pr view 262` は `mergeable: CONFLICTING`、`mergeStateStatus: DIRTY` を返した。
  - `origin/main` は `718484a` から `d8a6228` へ進み、PR #259 と PR #261 が merge 済み。
  - PR #262 の head は `4aa7f0f` で、作成時 CI は成功していた。
- 推定原因:
  - PR #259 の document chat scope 変更、または PR #261 の generated docs / task 変更が、PR #262 の web / generated docs / task 差分と重なった。
- 根本原因:
  - 並行 PR が同じ document 管理 UI と generated inventory 周辺を変更し、PR #262 が最新 main を取り込んでいなかった。
- 対応方針:
  - 最新 `origin/main` を PR branch に merge し、main 側の新機能を保持しながら PR #262 の共有 group selector 差分を残す。
  - generated docs は手編集ではなく再生成で解消する。
  - conflict 解消後に対象 web 検証と CI を確認する。
