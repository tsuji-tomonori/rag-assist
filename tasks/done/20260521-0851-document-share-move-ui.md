# ドキュメント共有・移動 UI/API 実装

状態: done

## 背景

現行仕様の「フォルダは共有・検索範囲・管理権限の単位」「文書操作は権限で制御する」という設計を維持しつつ、右側カラム廃止、フォルダ操作のモーダル化、ファイル単位共有、ファイル移動を追加する。

## 目的

ドキュメント管理画面の右側カラムを廃止し、フォルダ操作と文書操作を同じ権限体系で扱える UI/API/権限計算を実装する。

## 受け入れ条件

- AC-DOC-UI-001: ドキュメント管理画面に右側カラムが表示されない。
- AC-DOC-UI-002: フォルダ情報、フォルダ共有、フォルダ編集は、それぞれボタン押下でモーダル表示される。
- AC-DOC-ACTION-001: ドキュメント一覧の操作列に共有ボタンが表示される。
- AC-DOC-ACTION-002: ドキュメント一覧の操作列に移動ボタンが表示される。
- AC-DOC-SHARE-001: ファイル単位で user / group に readOnly / full を付与できる。
- AC-DOC-SHARE-002: ファイル単位共有はフォルダ権限への追加共有として扱われる。
- AC-DOC-SHARE-003: ファイル単位共有では、フォルダ由来の権限を打ち消さない。
- AC-DOC-SHARE-004: ファイルだけ共有されたユーザーは、そのファイルを閲覧・RAG参照できる。
- AC-DOC-SHARE-005: ファイルだけ共有されたユーザーは、元フォルダや同一フォルダ内の他ファイルを閲覧できない。
- AC-DOC-MOVE-001: 対象ファイルに full、移動先フォルダに full がある場合、ファイルを移動できる。
- AC-DOC-MOVE-002: 移動先フォルダに full がない場合、ファイル移動はできない。
- AC-DOC-MOVE-003: ファイル移動後、直接共有設定は維持される。
- AC-DOC-MOVE-004: ファイル移動後、フォルダ継承権限は移動先フォルダのものに変わる。
- AC-DOC-MOVE-005: ファイル移動後、古いフォルダ検索範囲ではそのファイルが検索されない。
- AC-DOC-AUDIT-001: ファイル共有変更とファイル移動は監査ログに残る。

## ドキュメント保守計画

API・権限・UI フローが変わるため、関連する `docs/` または既存 API 仕様ドキュメントを検索し、最小限の更新先を決める。該当する durable docs がない場合は、作業レポートに「更新不要または未発見」の理由を残す。

## PR レビュー観点

- `DocumentShareGrant` がフォルダ権限を打ち消さない加算方式になっていること。
- フォルダ権限がないユーザーへ元フォルダや兄弟ファイルが漏れないこと。
- 移動後に古い folderId の検索結果が残らないこと。
- 共有・移動 API が `reason` と適切な permission を要求すること。
- 本番 UI に未実装の見せかけ操作や固定データが混入していないこと。

## リスク

- 変更範囲が UI、API、store、RAG 検索にまたがるため、既存の抽象に合わせた段階実装が必要。
- 永続化層がローカル実装中心の場合、DynamoDB などの本番向け store 追加が別途必要になる可能性がある。
- E2E は環境依存のため、実行できない場合は理由を明記する。

## 実装メモ

- API は `DocumentShareGrant` ledger、実効文書権限、ファイル共有 API、ファイル移動 API、RAG/list の直接共有反映を追加した。
- UI は右側カラムを削除し、フォルダ情報・共有・名変更・移動・アップロードをモーダルへ移した。文書行には詳細・共有・移動・削除系操作を配置した。
- Web の旧右ペイン前提テスト 29 件は新仕様と矛盾するため skip に変更し、新 UI のモーダル/文書共有/文書移動テストを追加した。恒久的には旧テストの観点を新モーダル単位へ再分解する必要がある。
- Durable docs は該当更新先をこの作業中に特定できなかったため、API 契約は既存 OpenAPI contract test と route-level policy test で確認した。
- 追加レビュー対応で、共有設定取得を full 限定にし、`tenantId` 境界、文書実効権限ベース route metadata、一覧 capabilities、direct grant 削除 UI、移動時同名衝突 UI を追加した。
- 旧右ペイン前提テスト 29 件は skip ではなく todo に変更し、権限/capability の重要観点は新規 UI テストへ移した。
