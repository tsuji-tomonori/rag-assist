# mmrag-docqa-v1 全量テストケース実施

## 背景

`mmrag-docqa-v1` のテストケース実行が本来の 1091 件全量ではなく、一部件数に制限されている可能性がある。

## 目的

`mmrag-docqa-v1` において、本来の 1091 件全量を実施するよう修正する。

## スコープ

- `mmrag-docqa-v1` のテストケース件数を制限している実装・設定の調査
- 1091 件全量を実施するためのコードまたは設定の修正
- 関連するテスト・検証の実行
- 必要に応じたドキュメント更新

## 計画

1. `mmrag-docqa-v1` の参照箇所とテストケース生成・実行設定を調査する。
2. 件数制限の原因を特定し、全量 1091 件を既定で対象にする。
3. 変更範囲に応じた最小十分な検証を実行する。
4. 作業レポート、commit、push、PR、受け入れ条件確認コメント、セルフレビューコメントまで完了する。

## ドキュメント保守計画

実装挙動や運用手順に影響する記述が README または docs に存在する場合は更新する。該当記述がない場合は、レポートと最終回答に「不要」として理由を記載する。

## 受け入れ条件

- `mmrag-docqa-v1` の通常実行で、意図的なサンプル制限により 1091 件未満にならない。
- 全量件数 1091 件を前提にしたテストケース数の検証が追加または既存検証で確認できる。
- 変更範囲に見合うテストまたは静的検証が pass している。
- 未実施の検証がある場合は、理由と残リスクを PR 本文・コメント・作業レポートに明記する。

## 検証計画

- 変更ファイル確認: `git diff --name-only`
- 差分検査: `git diff --check`
- 対象テスト: 変更箇所に対応する npm/test/task コマンドを選定して実行する

## PR レビュー観点

- 1091 件全量を既定で対象にしているか
- benchmark 期待語句・QA sample 固有値・dataset 固有分岐を実装へ混入していないか
- docs と実装に不整合がないか
- 変更範囲に見合う検証が行われているか

## リスク

- 全量実行自体は時間や外部サービス依存が大きい可能性があるため、環境上実行できない場合はテストケース数の静的・単体検証で補完する。
