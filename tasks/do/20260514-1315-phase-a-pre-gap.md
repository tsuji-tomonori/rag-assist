## 背景

`.workspace/wsl-localhost-ubuntu-home-t-tsuji-proje-vivid-cloud.md` の移行計画では、巨大な章別仕様書を canonical docs として据える前に、各 Phase の先頭で既存仕様・実装との差分と踏襲すべき既存挙動を明示することが求められている。
Phase A は後続 Phase B-J/G の前提となる docs canonical 化と章から REQ へのトレース整備を扱うため、まず現行 `docs/`、`docs/spec-recovery/`、API/Web/contract の影響範囲を調査する。

## スコープ

- 含む:
  - 仕様章: 0/1/1A/2/3/6/24 章を中心に、Phase A の docs canonical 化に必要な章構成を確認する。
  - 影響しうる docs: `docs/spec-recovery/`, `docs/1_要求_REQ/`, `docs/REQUIREMENTS.md`, `docs/ARCHITECTURE.md`
  - 影響しうる実装: `apps/api/src/`, `apps/web/src/`, `packages/contract/src/` の現行構成の棚卸し
  - 後続タスク: A1/A2/A3 の実施範囲、scope-out、リスク抽出
- 含まない:
  - Phase B 以降の認可・品質・リネーム・非同期エージェント実装
  - 章別仕様書全文の canonical コピー
  - 既存未追跡ディレクトリの削除または整理
