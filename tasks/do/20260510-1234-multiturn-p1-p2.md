# Multi-turn benchmark P1 本格実装 / P2 retrieval reranking 強化

状態: do

## 背景

P0/P1 で conversation-aware benchmark と query trace の土台はできているが、P1 は deterministic trace に留まり、検索実行で standalone query / retrieval query 群 / previous citation prior を十分に使えていない。
また、MTRAG / ChatRAG Bench 系で正しい page / chunk を取るには reranking と page-aware retrieval の改善が必要になる。

## 目的

conversation-aware query rewriting を検索実行へ反映し、BM25 + dense + RRF の候補集合に対して page/layout-aware feature と previous citation prior を使った reranking を追加する。

## スコープ

- `decontextualizedQuery.retrievalQueries` を embedding / evidence search の query 群へ反映
- previous turn citation の document / page / chunk を conversation state に保持
- agent `rerank_chunks` への page/layout-aware feature 追加
- previous turn citation document/page soft prior
- P1/P2 の debug trace と unit test 追加
- expected page hit / MRR への効果測定導線を壊さない

## ドキュメント保守計画

API schema 自体は既存 `conversationHistory` / `conversation` の範囲内で変更し、外部 contract を広げる場合だけ docs / OpenAPI を更新する。
今回は内部検索・rerank ロジック中心のため、README などの durable docs 更新は必要性を確認して判断する。

## 受け入れ条件

- [x] conversation-dependent turn で standalone query / retrieval query 群が evidence search に使われる。
- [x] previous turn citation の document / page / chunk 情報が conversation state に残る。
- [x] page / section / heading / table kind などの metadata を rerank feature として使える。
- [x] previous turn citation prior は soft boost に留まり、根拠なし回答を許さない。
- [x] benchmark 固有 row id / expected answer 分岐を入れていない。
- [x] debug trace / unit test で P1 本格化と P2 rerank feature を確認できる。
- [x] 既存 single-turn retrieval の regression を検証している。

## 検証計画

- API agent node unit test
- benchmark unit test または benchmark smoke
- API typecheck / test
- `git diff --check`

## PR レビュー観点

- 会話履歴の assistant answer を根拠として直接使っていないか。
- previous citation prior が同一文書・近傍 page の soft boost に留まっているか。
- benchmark 固有の expected answer / expected page / row id を実装ロジックへ入れていないか。
- single-turn の検索・回答選択が過度に変わっていないか。

## リスク

- deterministic rewrite のため、LLM rewrite ほどの自然言語補完精度はない。
- rerank feature の重みが強すぎると、会話履歴の prior が新しい topic shift を妨げる可能性がある。

## 実施内容

- `decontextualizedQuery.retrievalQueries` が memory clue 生成後も `expandedQueries` に残るようにした。
- conversation benchmark runner が前 turn の assistant citation を `conversation.turns` に引き継ぐようにした。
- `Citation` / conversation citation に optional `pageStart` / `pageEnd` を追加し、OpenAPI docs を再生成した。
- conversation state に previous citation の document / file / chunk / page range を保持するようにした。
- agent rerank と search `cheapRerank` に heading / section / chunk kind / page continuity / previous citation prior の lightweight feature boost を追加した。
- P1/P2 の挙動を確認する unit test を追加した。
