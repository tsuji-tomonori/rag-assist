# チャットUI参照画像寄せ

状態: do

## 背景

ユーザーから、参照画像「社内QAチャットボットエージェント」のような見た目にする依頼があった。

## 目的

`memorag-bedrock-mvp` のチャット画面を、白基調・左ナビ・中央チャットカード・参照元表示・入力欄を備えた業務向けチャットUIへ近づける。

## スコープ

- 対象: `memorag-bedrock-mvp/apps/web` のチャット画面、必要最小限のアプリシェルCSS。
- 非対象: API契約、認証・認可、RAG回答ロジック、データ永続化の変更。

## 計画

1. 既存のチャットコンポーネントとCSS構成を確認する。
2. 画像に合わせてチャットカード、メッセージ、参照元、候補ボタン、入力欄を調整する。
3. 必要に応じて左ナビと上部バーの見た目を整える。
4. モバイル幅でレイアウト崩れが起きないよう調整する。
5. Web向けのテスト、typecheck、差分チェックを実行する。
6. 作業レポート、commit、PR、受け入れ条件コメント、セルフレビューコメントまで実施する。

## ドキュメント保守計画

APIや運用手順は変更しないため、恒久ドキュメント更新は原則不要と判断する。作業内容と判断は `reports/working/` の作業レポートに記録する。

## 受け入れ条件

- 参照画像に近い白基調の業務向けチャットUIになっている。
- 既存のチャット送信、添付、回答表示、参照元表示の機能を壊していない。
- デスクトップとモバイルで入力欄、回答カード、ボタン、テキストが重ならない。
- 変更範囲に見合う Web のテストと typecheck が成功している。
- 作業レポート、PR本文、受け入れ条件確認コメント、セルフレビューコメントに未実施検証とリスクを正直に記載している。

## 検証計画

- `git diff --check`
- `npm --prefix memorag-bedrock-mvp run typecheck -w @memorag-mvp/web`
- `npm --prefix memorag-bedrock-mvp run test -w @memorag-mvp/web`
- 必要に応じて `npm --prefix memorag-bedrock-mvp run build -w @memorag-mvp/web`

## PRレビュー観点

- 参照画像に寄せる変更がチャット画面の責務内に収まっていること。
- CSS変更が他画面へ過剰に波及していないこと。
- アクセシビリティ上、ボタン名・フォーカス・タッチターゲットを悪化させていないこと。
- RAGの根拠性、認可境界、API契約を変更していないこと。

## リスク

- visual regression snapshot は環境依存で更新が必要になる可能性がある。
- 画像との差分は厳密なピクセル一致ではなく、既存アプリの機能制約に合わせた近似になる。
