## 背景

Wave 4-pre の `J1-pre-gap` として、仕様 14B/21A 章と現行 API 実装・OpenAPI 関連ドキュメントの差分を調査する。コード変更は原則行わず、`GET /openapi.json` を runtime source of truth とする現状、生成 Markdown の派生物性、CI drift gate、API lifecycle / compatibility / deprecation の現状を整理する。

## 目的

- 仕様 14B/21A 章と `GET /openapi.json`、OpenAPI 生成・docs quality/drift gate、API lifecycle のギャップを `confirmed` / `partially covered` / `missing` / `divergent` で記録する。
- 後続 `J1-openapi-runtime-source` の scope / scope-out を明確化する。
- 仕様対応表と関連 REQ / spec-recovery docs を必要最小限で更新する。

## スコープ

- 主対象:
  - `docs/spec/gap-phase-j1.md`
  - `docs/spec/CHAPTER_TO_REQ_MAP.md`
  - 関連 REQ / `docs/spec-recovery/`
  - `reports/working/`
  - この task md
- 読み取り対象:
  - API route / schema / OpenAPI 生成コード
  - docs quality / drift gate の CI・script・package script
- 原則 scope-out:
  - API runtime コード変更
  - OpenAPI 生成物の再生成
  - CI workflow の変更
  - API lifecycle 実装の追加

## 作業計画

1. 仕様 14B/21A、OpenAPI、docs gate、API lifecycle に関係する docs / scripts / tests を棚卸しする。

3. `docs/spec/gap-phase-j1.md` を作成または更新し、既存挙動と後続 `J1-openapi-runtime-source` の scope / scope-out を記録する。
4. `docs/spec/CHAPTER_TO_REQ_MAP.md` と関連 REQ / traceability を必要最小更新する。
5. 指定検証を実行し、作業レポートを残す。
