# Web UI a11y メタデータ整備

## 背景

Web UI インベントリ生成を拡張し、スクリーンリーダー利用者向けに必要な accessible name / description / state を方針化し、UI と仕様書へ反映したい。

## 目的

- どの要素に a11y メタデータを付けるべきかを repository local skill とセルフレビュー観点に追加する。
- 方針に従って Web UI 全体の操作要素・状態要素へ日本語の accessible name / description / state を整備する。
- 静的解析ツールで a11y 情報を抽出し、生成仕様書へ記載する。

## スコープ

- `skills/` 配下への Web a11y metadata skill 追加。
- `skills/pr-review-self-review/SKILL.md` の Web UI self-review 観点追加。
- `memorag-bedrock-mvp/apps/web/src` の主要 UI コンポーネントの a11y 属性整備。
- `tools/web-inventory/generate-web-inventory.mjs` の a11y 抽出・Markdown 出力拡張。
- 生成 docs の更新。

## 対象外

- 画面デザイン変更。
- 実行時 Playwright + screen reader による全操作検証。
- API / RAG runtime / infra の挙動変更。

## 受け入れ条件

- [x] どの要素に a11y メタデータを付けるべきかが skill に定義されている。
- [x] PR セルフレビュー観点に Web a11y metadata の確認項目が追加されている。
- [x] 主要な操作可能要素、アイコンのみ操作、危険操作、状態付き UI、フォーム項目に必要な日本語 accessible name / description / state が付いている。
- [x] 静的解析ツールが a11y 属性と推定 accessible name / description / status を抽出する。
- [x] 生成仕様書に a11y 情報と不足・注意事項が記載される。
- [x] 実行した検証と未実施の制約を report / PR に明記する。
