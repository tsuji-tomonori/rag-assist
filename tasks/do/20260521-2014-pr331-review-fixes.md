# PR331 レビュー指摘修正

状態: doing

## 背景

PR #331 `doc-ui-share-move` に対して Request changes が出ている。指摘は UI レイアウト変更だけではなく、ファイル直接共有、tenant 境界、文書実効権限、RAG 検索範囲、行操作可否、テスト skip/todo 精査を含む認可モデル変更として扱う必要がある。

## 目的

P0/P1 指摘を優先して、ファイルだけ共有されたユーザーへの情報漏えい防止、DocumentShareGrant の tenant 境界、文書実効権限ベースの API/UI 判定、直接共有削除、権限不変条件テストの復活を実装・検証する。P2 と E2E/Actions は実施可能範囲を確認し、未実施なら理由を明記する。

## タスク種別

修正

## なぜなぜ分析サマリ

- 問題文: PR #331 はファイル共有・移動により文書単位の認可経路を追加したが、review では readOnly 直接共有ユーザーへの共有情報漏えい、tenant 境界不足、folder/group 権限前提の route metadata、権限不変条件テストの todo 化が指摘された。
- 確認済み事実:
  - PR は `GET/PUT /api/documents/{documentId}/share` と `POST /api/documents/{documentId}/move`、Web の文書共有/移動 UI、RAG 検索経路を変更している。
  - `apps/web/src/features/documents/components/DocumentWorkspace.test.tsx` には 29 件の `it.todo` が残っている。
  - 直近の PR 本文では一部指摘対応済みと記載されているが、コード実態とテスト網羅の再確認が必要。
- 推定原因:
  - 初期実装ではフォルダ共有モデルを文書共有へ拡張する際、既存の folder/group 権限前提や右ペイン前提テストを十分に置換できていなかった。
  - CI 通過を優先した過程で、旧 UI と矛盾するテストの todo 化に権限不変条件まで混在した可能性がある。
- 根本原因:
  - ファイル単位共有を「UI 追加」として進めた結果、API response の最小化、tenant boundary、route metadata、row capabilities、RAG scope まで含む認可モデル変更としての横断テストが不足した。
- 対策:
  - API/store/service/schema/static policy を文書実効権限と tenant 境界で再確認し、不足があれば修正する。
  - Web 行操作とモーダル操作を backend capabilities と対象リソース権限で判定する。
  - 旧右ペイン DOM 依存テストと権限不変条件テストを分離し、後者は現行 UI に合わせて復活させる。
  - 受け入れ条件ごとに unit/API/UI test を追加または確認し、未実行 E2E/Actions は未検証として明示する。
