# PR331 競合・CI 修正

- 状態: in_progress
- タスク種別: 修正
- 対象 PR: https://github.com/tsuji-tomonori/rag-assist/pull/331

## 背景

PR #331 は `codex/document-share-move-ui` から `main` への文書共有・移動 UI/API 追加 PR。
GitHub 上で `mergeable=false` となっており、MemoRAG CI Result でも web lint、OpenAPI docs check、web inventory check、api/web coverage test が失敗している。

## 目的

PR ブランチを最新 `origin/main` に追従させ、競合と CI 失敗要因を解消する。

## スコープ

- `origin/main` 取り込み時の競合解消
- CI 失敗項目に対応する実装、生成 docs、テスト期待値の修正
- 必要な最小十分の検証
- 作業レポート、PR コメント、task 状態更新

## なぜなぜ分析サマリ

- 問題文: PR #331 の head `1714160a` は `origin/main` に対して mergeable ではなく、CI で web lint、OpenAPI docs freshness、web inventory freshness、api/web coverage が失敗している。
- 確認済み事実:
  - GitHub Apps の PR 取得結果で `mergeable=false`。
  - ユーザー提示の CI 結果で `npm exec -- eslint apps/web ...`、`npm run docs:openapi:check`、`npm run docs:web-inventory:check`、api/web coverage が failure。
  - PR ブランチは文書共有・移動 API/UI と生成 docs を広く変更している。
- 推定原因:
  - `origin/main` 側の同領域変更と PR #331 の差分が衝突している。
  - OpenAPI と web inventory は生成物のため、main 取り込み後に再生成が必要。
  - coverage/lint は main 追従後のテスト期待値・未使用コード・分岐追加により閾値または lint 条件が変化している可能性がある。
- 未確定点:
  - 具体的な競合ファイルと各 CI failure の直接原因は merge とローカル再実行で確定する。
- 根本原因:
  - PR #331 の長い feature branch が、文書管理・認可・生成 docs 周辺で更新が続く `main` に追従できていない。
- 対応方針:
  - 最新 `origin/main` を取り込み、衝突箇所を PR #331 の意図と main の最新仕様の両方を保持する形で解消する。
  - 生成 docs は再生成し、coverage/lint failure は実際の failure 出力に基づいて修正する。
