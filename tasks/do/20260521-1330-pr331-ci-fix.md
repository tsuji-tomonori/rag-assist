# PR331 CI 修正

状態: doing

## 背景

PR #331 `doc-ui-share-move` の MemoRAG CI で、web lint、OpenAPI contract docs、web inventory、api/web coverage test が失敗している。PR は draft のまま、ready 化前に CI 失敗を解消する必要がある。

## 目的

PR #331 の CI 失敗原因を特定し、実装・生成物・テスト期待値を修正して、少なくとも報告された失敗コマンドをローカルで pass させる。

## タスク種別

修正

## なぜなぜ分析サマリ

- 問題文: PR #331 の CI で、UI/API 実装差分に対する lint、OpenAPI docs freshness、web inventory freshness、coverage test が期待状態から外れ、merge 判定に進めない。
- 確認済み事実:
  - CI 報告では `npm exec -- eslint apps/web ...`、`npm run docs:openapi:check`、`npm run docs:web-inventory:check`、api/web coverage test が fail。
  - 直前のローカル検証では typecheck と通常 test は pass していたが、lint、docs freshness、coverage gate は同じ範囲で未実行。
  - PR は API route/schema と Web UI/API client を変更しており、生成ドキュメント・inventory・coverage 閾値に影響し得る。
- 推定原因:
  - 新規/変更コードに lint rule 違反または stale inventory が残っている。
  - OpenAPI runtime source 変更後に生成済み docs が更新されていない。
  - coverage test は todo/skip 化や未カバー分岐の追加により閾値未達、または失敗テストがある。
- 根本原因:
  - 変更範囲に対する CI 同等コマンドの選択が不足し、通常 test/typecheck だけで pass 判断したため、生成物 freshness と coverage gate の回帰を検出できなかった。
- 対策:
  - 報告された CI コマンドをローカルで再現し、失敗ログに基づいてコード、テスト、生成物を修正する。
  - 修正後は同じコマンドを再実行し、未実行チェックは未実行として PR コメント/レポートに明記する。
