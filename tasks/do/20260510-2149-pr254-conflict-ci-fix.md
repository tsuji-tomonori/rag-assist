# PR254 競合解決とCI修正

状態: do

## 背景

PR #254 は `mergeStateStatus: DIRTY` となり、MemoRAG CI では web test が failure と報告された。`origin/main` には PR #253 の `document-url-state` 系変更が merge 済みであり、PR #254 の同種実装と競合している。

## 目的

PR #254 を最新 `origin/main` と競合なく統合できる状態にし、MemoRAG CI の web test failure を再発しない形に修正する。

## タスク種別

修正

## なぜなぜ分析サマリ

- 問題文: PR #254 が GitHub 上で競合状態になり、MemoRAG CI の web test が failure として報告された。
- 確認済み事実:
  - `gh pr view 254` は `mergeStateStatus: DIRTY` を返した。
  - CI comment は web test failure を示した。
  - `origin/main` は PR #253 `codex/document-url-state` を merge 済みで、同じ文書管理 URL state 領域を変更している。
  - PR #254 branch の local working tree は clean。
- 推定原因:
  - PR #254 作成後に main 側へ同種の URL state 実装が入り、同一ファイルの差分が衝突した。
  - CI failure は古い PR merge ref 上で web coverage test が失敗したもので、競合解消後に最新 main 基準で再検証が必要。
- 根本原因:
  - 並行 PR が同じ `DocumentWorkspace` / app shell routing 領域を変更し、PR #254 が main の後続変更を取り込んでいなかった。
- 対応方針:
  - 最新 `origin/main` を PR #254 branch に取り込み、重複実装を main 側へ寄せて競合を解消する。
  - web coverage failure を local で再現し、実装または test expectation の不整合を修正する。
  - 対象検証を再実行し、PR に受け入れ条件とセルフレビューを更新する。
