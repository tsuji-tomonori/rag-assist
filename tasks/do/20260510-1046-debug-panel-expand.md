# DebugPanel 拡大ボタンの実装

## 背景

Web UI の `DebugPanel` に「デバッグパネルを拡大表示」ボタンがあるが、押下時の処理がなく、利用者にとって反応しない操作になっている。

## 目的

`DebugPanel` の拡大ボタンを実際に動作する UI にし、デバッグトレースを広い領域で確認できるようにする。

## スコープ

- `memorag-bedrock-mvp` の Web UI 実装
- `DebugPanel` に関係するスタイルとテスト
- 作業完了レポート

## スコープ外

- debug trace API の変更
- 権限・認可ロジックの変更
- URL ルーティングや citation drawer など P1 以降の改善

## 作業計画

1. `DebugPanel` の現行実装とテストを確認する。
2. 拡大表示の状態管理、モーダルまたは dialog 相当の UI、閉じる操作を実装する。
3. キーボード操作と accessible name を確認する。
4. 既存テストに押下時の挙動を追加する。
5. 変更範囲に見合う検証を実行する。
6. 作業レポートを作成し、commit/push/PR/comment まで進める。

## ドキュメント保守方針

ユーザー可視の軽微な UI バグ修正であり、公開 API、運用手順、要求仕様の変更は想定しない。恒久ドキュメント更新が不要と判断した場合は、作業レポートと PR 本文に理由を記録する。

## 受け入れ条件

- [x] 「デバッグパネルを拡大表示」ボタンを押すと、デバッグパネルの拡大表示が開く。
- [x] 拡大表示は閉じるボタンまたはキーボード操作で閉じられる。
- [x] 拡大表示内でも trace 情報が読み取れる。
- [x] ボタンが無反応な UI として残らない。
- [x] 対象 UI の挙動をテストまたは同等の検証で確認する。
- [x] 変更に伴う不要な mock/fake product data を追加しない。

## 検証計画

- Web UI の対象テストを実行する。
- TypeScript 変更に対して web typecheck または同等の検証を実行する。
- `git diff --check` を実行する。

## PR レビュー観点

- Debug trace や内部情報の露出範囲を広げていないこと。
- 操作可能な UI に accessible name と閉じる導線があること。
- 変更範囲に見合うテストがあること。
- 恒久 docs 更新が不要である判断が妥当であること。
