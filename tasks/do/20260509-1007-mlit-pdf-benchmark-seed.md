# MLIT PDF 図表 RAG ベンチマーク seed 取り込み

状態: doing

## 背景

ユーザーから、`.workspace/mlit_pdf_figure_table_rag_benchmark_seed.xlsx` について「CSV/JSONL に変換して管理するか判断 go」と依頼された。XLSX は国土交通省系 PDF の図表 RAG 評価用 seed データであり、リポジトリ内の評価資産として再利用しやすい形式にする必要がある。

## 目的

XLSX の内容と既存リポジトリ構成を確認し、管理形式を判断したうえで、必要なデータファイルと利用手順を追加する。

## スコープ

- XLSX シート構成と行・列の確認
- 既存 benchmark / docs / dataset 配置の確認
- XLSX 原本、CSV、JSONL の管理方針決定
- 必要な変換ファイルと README の追加
- 変更範囲に応じた検証
- 作業レポート、commit、PR、受け入れ条件確認コメント

## 作業計画

1. XLSX と既存 benchmark 構成を調査する。
2. 管理形式を決める。
3. データファイルと README を追加する。
4. 変換結果の整合性を検証する。
5. 作業レポートを作成する。
6. commit / push / PR / PR コメントまで実施する。

## ドキュメント保守計画

ベンチマーク seed の利用者が投入・採点手順を確認できるよう、データ配置先に README を追加する。既存 docs への追記は、既存構成に関連 index がある場合のみ最小限に行う。

## 受け入れ条件

- [x] XLSX のシート構成、主要列、レコード数を確認している。
- [x] CSV/JSONL で管理するかどうかの判断理由を README または作業レポートに記載している。
- [x] ベンチマーク seed データがリポジトリ内の適切な場所に配置されている。
- [x] 利用手順、出典、採点観点、既知の制約を文書化している。
- [x] 変換結果の行数・必須列・ID 重複を検証している。
- [x] 変更範囲に見合う検証を実行し、未実施項目があれば理由を記録している。
- [x] 作業完了レポートを `reports/working/` に作成している。
- [ ] PR 作成後、受け入れ条件確認コメントとセルフレビューコメントを日本語で記載している。

## 検証計画

- `python3` による XLSX/CSV/JSONL の行数・列・ID 整合性確認
- `git diff --check`
- 変更がデータ・Markdown 中心のため、アプリケーション単体テストは原則対象外。必要性が見つかった場合のみ追加する。

## PR レビュー観点

- データ形式が benchmark runner や将来の評価スクリプトから扱いやすいか。
- 出典 URL と評価上の制約が明確か。
- 実施していない検証を実施済みとして書いていないか。
- 固有 benchmark の期待語句をアプリ実装へ混入させていないか。

## リスク

- seed データの gold answer は建築実務・法務・省エネ適判の専門レビュー未実施である可能性がある。
- `.workspace` の XLSX は作業入力であり、リポジトリの永続成果物とは別扱いにする必要がある。
